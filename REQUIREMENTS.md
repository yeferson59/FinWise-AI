# Informe de implementación

## Prototipo Funcional:

Versión inicial del software que implementa las funcionalidades básicas.
Descripción detallada de cómo se desarrolló el prototipo, descripción de módulos, librerías, y componentes en general
Explicación de cómo se utilizaron las tecnologías y herramientas específicas.
Puede ser una versión simplificada, pero funcional, para demostrar la futura aplicación de la inteligencia artificial.

## Código fuente

Repositorio de código (puede ser en una plataforma como GitHub). El código debe estar documentado para facilitar la comprensión.

## Pruebas de software

Pruebas Iniciales: Plan de pruebas básico.
Informe de resultados de las pruebas realizadas.
Identificación y documentación de posibles problemas.

# Indicaciones en canto a la presentación de la sección

El propósito de esta sección es documentar el proceso de construcción y validación del prototipo funcional. Se debe escribir en estilo técnico, objetivo y con redacción impersonal, evitando el uso de la primera persona. Se recomienda redactar en tiempo pasado, aunque las instrucciones aquí se expresan en infinitivo para indicar las acciones requeridas.

## 3.1. Prototipo funcional

## Indicaciones de fondo:

- Describir cómo se materializó la solución propuesta durante la fase de diseño.
- Explicar las funcionalidades implementadas y la estructura general del sistema.
- Detallar las etapas de desarrollo del prototipo, los componentes principales y las tecnologías utilizadas.
- Incluir información sobre el entorno de desarrollo (lenguaje, librerías, frameworks, versiones y hardware empleado).
- Presentar los módulos implementados (interfaz, lógica, inteligencia artificial, base de datos, comunicación entre componentes, etc.).
- Mostrar capturas de pantalla del sistema en ejecución o diagramas de flujo de interacción.
- Ejemplificar el funcionamiento mediante un caso de uso: entrada, procesamiento y salida de resultados.
- Mencionar las iteraciones de mejora o versiones previas si existieron.

## Indicaciones de forma:

- Redactar entre 2 y 3 páginas.
- Organizar el contenido con subtítulos que identifiquen los módulos o componentes principales.
- Incluir figuras numeradas con pie de imagen descriptivo. Ejemplo: Figura 1. Interfaz del prototipo funcional.
- Utilizar párrafos breves, claros y precisos, con lenguaje técnico y coherente.

## 3.2. Código fuente

## Indicaciones de fondo:

- Describir la estructura general del código fuente sin incluirlo completamente en el documento.
- Explicar la organización de carpetas y archivos del proyecto, indicando el propósito de cada módulo o script.
- Listar las dependencias y librerías externas empleadas.
- Referenciar el repositorio donde se aloja el código (GitHub, GitLab u otro) e indicar el método de acceso.
- Incluir instrucciones breves para ejecutar el sistema (comando principal o script inicial).
- Mencionar el uso de control de versiones y buenas prácticas de documentación del código.

## Indicaciones de forma:

- Presentar la estructura de directorios mediante una lista jerárquica o diagrama. Incluir fragmentos breves de código (máximo 10–15 líneas) cuando sea necesario ilustrar la lógica principal.
- Agregar comentarios en el código para facilitar su comprensión.
- Citar la fuente del repositorio siguiendo normas APA si se trata de una entrega formal.

## 3.3. Pruebas de software

## Indicaciones de fondo:

- Documentar las pruebas realizadas para validar el funcionamiento del prototipo.
- Definir el plan de pruebas, especificando qué componentes se evaluaron y qué herramientas se utilizaron.

Describir los tipos de pruebas efectuadas:

- Pruebas funcionales (verificar que cada función cumpla su propósito).
- Pruebas de integración (validar la comunicación entre módulos).
- Pruebas de rendimiento (evaluar tiempos de ejecución o uso de recursos).
- Pruebas de inteligencia artificial (analizar métricas de precisión, recall, F1-score, etc.).
- Pruebas de seguridad (evaluar vulnerabilidades y amenazas).
- Presentar los resultados de las pruebas mediante tablas, gráficas o matrices de confusión.
- Analizar los resultados obtenidos, indicando fortalezas, debilidades y comportamientos inesperados.
- Describir los ajustes realizados tras la ejecución de las pruebas.
- Concluir con una valoración general del desempeño del sistema y posibles mejoras.

## Indicaciones de forma:

- Utilizar tablas y figuras numeradas con títulos descriptivos. Ejemplo: Tabla 2. Resultados de las pruebas funcionales.
- Redactar con precisión, sin juicios personales ni opiniones subjetivas.
- Mantener un tono formal, técnico y analítico.
- Finalizar la sección con una síntesis que relacione los resultados con los objetivos planteados en la fase de análisis.

## Otras recomendaciones

- Redactar todo el informe con un enfoque técnico, coherente y secuencial.
- Usar conectores lógicos para enlazar ideas: posteriormente, a continuación, como resultado, en consecuencia, finalmente.
- Evitar expresiones en primera persona (yo, nosotros, mi proyecto).
- Utilizar verbos en voz pasiva refleja o impersonal: se implementó, se configuró, se validó.
- Mantener la uniformidad en el estilo de figuras, tablas y citas.
- Gestionar las referencias bibliográficas mediante normas APA.
